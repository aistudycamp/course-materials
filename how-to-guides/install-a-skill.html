How-To Guide

How to Download and Install a Skill

Get a shared skill onto your machine and into Claude Code so it's available in every session.

What's a Skill?

A skill is a specialized workflow Claude knows how to run — like brainstorming, reviewing code, or drafting a PRD. Someone writes the recipe once, you install it, and then you can trigger it by name (e.g. /brainstorm) or just by describing what you want. Claude follows the recipe instead of winging it.

Skills live in ~/.claude/skills/ on your computer. You can install them two ways: through Claude Code's built-in plugin marketplace (easy, no terminal), or by cloning a GitHub repo directly (for skills that aren't packaged as plugins).

Prerequisites

  • Claude Code installed and working — see Install Claude Code
  • A skill to install — either a plugin marketplace link or a GitHub repo from your instructor

Option A: Install from a plugin marketplace (easiest)

If the skill is packaged as a plugin, you can install it from inside Claude Code — no terminal, no cd, no git. Start Claude Code in any project, then run these two slash commands:

1

Add the marketplace

Inside Claude Code, type:

claude code
/plugin marketplace add owner/repo

Replace owner/repo with the GitHub path your instructor gave you (e.g. tiobenito/aisc).

2

Install the plugin

Then run /plugin to open the plugin picker, or install directly:

claude code
/plugin install skill-name@marketplace-name

The marketplace name usually matches the repo. Claude will confirm when it's installed.

Option B: Install from a GitHub repo

Some skills aren't packaged as plugins — they're just a folder with a SKILL.md inside. For those, you drop the folder into ~/.claude/skills/ once and you're done.

1

Get the folder onto your machine

Either clone the repo in your terminal:

zsh
$ git clone https://github.com/tiobenito/aisc

…or, if you'd rather skip git: open the repo in your browser, click the green Code button, choose Download ZIP, and unzip it. You'll end up with the same folder either way.

2

Find the folder that contains SKILL.md

Inside the download, look for a file named exactly SKILL.md. The folder holding that file is the skill. Example: in the aisc repo, the skill-builder lives at aisc/skill-builder/SKILL.md, so skill-builder is the folder you want.

3

Copy that folder into ~/.claude/skills/

One command — this creates the skills directory if needed, then copies the folder in:

zsh
$ mkdir -p ~/.claude/skills && cp -r aisc/skill-builder ~/.claude/skills/

That's the whole install. No restart required — Claude Code picks up new skills automatically next time you invoke one.

What "working" looks like

You'll know it worked when you can see the skill from inside Claude Code. Start Claude Code in any project and type a forward slash:

claude code
/ /brainstorm — Interactive design exploration /skill-builder — Turn a workflow into a skill …

Your new skill should appear in the autocomplete list. You can also just describe the task in plain English — if the skill's trigger matches, Claude will run it.

Tip

To double-check the install from the terminal: ls ~/.claude/skills/ should list your skill's folder, and inside it you should see SKILL.md.

Troubleshooting

Skill doesn't show up in / autocomplete

The folder must be directly inside ~/.claude/skills/, not nested an extra level. The path should be ~/.claude/skills/[skill-name]/SKILL.md.

"No skill found"

The file must be named exactly SKILL.md — capital S, capital MD. Rename it if needed.

Skill runs but behaves wrong

Open the SKILL.md file — it often lists prerequisites or setup steps you may have missed.

Permission error on copy

Try sudo cp -r ... and enter your password when prompted.
