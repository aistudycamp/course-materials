How-To Guide

How to Connect the Google Workspace CLI

Install the gws CLI and connect it to your Google accounts so you can read email, calendar, Drive, and Sheets from the terminal.

What This Is Good For

Once set up, gws gives Claude and your scripts direct access to your Google data — Gmail, Calendar, Drive, Sheets, Docs, and more. You can connect multiple accounts (personal + work) and switch between them with a single env var.

Safety

Always set this up with read-only scopes. Don't give Claude the ability to send email or modify your calendar via API — compose responses in the chat and send them yourself.

1

Install gws and gcloud

gws auth setup uses gcloud under the hood, so install both:

zsh
$ brew install googleworkspace-cli google-cloud-sdk $ gcloud auth login

gcloud auth login opens a browser once so Google can verify your identity. Come back to the terminal after.

2

Set up your first account

Run setup with the --readonly flag. Follow the browser prompts to pick your Google account and approve access:

zsh
$ gws auth setup --login --readonly

That's it. Credentials are saved locally and you don't need to understand what happened behind the scenes.

3

Add a second account

If you have both a personal and a work Google account, keep them in separate config directories. Run setup once per account, pointing at a different directory each time:

zsh
$ mkdir -p ~/.config/gws-personal ~/.config/gws-work $ GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-personal gws auth setup --login --readonly $ GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-work gws auth setup --login --readonly

Each account stores its own credentials in its own directory. They don't interfere with each other.

Important

Keep personal and work accounts strictly separated. Never point a work session at the personal config dir — it's an easy way to leak data across contexts.

4

Verify it works

List your 5 most recent emails to confirm auth succeeded:

zsh
$ GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-personal gws gmail users messages list --params '{"userId": "me", "maxResults": 5}'

If you see a list of message IDs, you're connected. If not, jump to Troubleshooting below.

Switching Accounts

Set GOOGLE_WORKSPACE_CLI_CONFIG_DIR before any gws command to pick the account. Add aliases to your ~/.zshrc so you don't retype it:

~/.zshrc
alias gwspersonal='GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-personal gws' alias gwswork='GOOGLE_WORKSPACE_CLI_CONFIG_DIR=~/.config/gws-work gws'

Then use gwspersonal gmail ... and gwswork calendar ....

A Couple of Useful Commands

zsh
# Search unread mail $ gws gmail users messages list --params '{"userId": "me", "q": "is:unread"}' # See which account is active $ gws auth status

Run gws --help to explore everything else.

Troubleshooting

"restricted_client" error during login

Make sure you used the --readonly flag. Read-only scopes avoid Google's unverified-app warning.

"gws: command not found"

Close and reopen your terminal after installing.

Wrong account used

Check GOOGLE_WORKSPACE_CLI_CONFIG_DIR in your shell. Run gws auth status to see which account is active.

Token expired

Re-run gws auth setup --login --readonly in the relevant config directory.
