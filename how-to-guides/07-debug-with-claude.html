Guide 07

How to Debug with Claude Code

How to describe problems so Claude can fix them — and build a project memory so the same bug doesn't bite twice.

The Debugging Pattern

Claude needs three things from you:

  1. What you expected to happen
  2. What actually happened (the error or wrong behavior)
  3. Where it happened (which file, which action)
1

Paste the error, don't paraphrase

Good:

claude
I'm getting this error when I click the submit button: TypeError: Cannot read properties of undefined (reading 'map') at RecipeList (src/components/RecipeList.tsx:14)

Bad: "It's broken."

2

Add context + point to the file

Tell Claude when it happens and where to look. Claude Code can read your files directly, so a path is enough.

claude
This error shows up when I click "Save Recipe" after filling out the form. It was working before I added the ingredients field. The error is in src/components/RecipeList.tsx — can you take a look?
3

Let Claude investigate, then review

Claude will read the relevant files and propose a fix. Review it before accepting.

Tip

If the fix doesn't work, say "that didn't fix it, here's what's still happening:" and share the new error. Don't just say "still broken."

The Template

When in doubt, use this format:

claude
I'm working on [feature/page]. When I [action], I expected [expected behavior]. Instead, I get [actual behavior/error]. The relevant file is [file path].

Teach Claude Your Common Errors

Once a bug is fixed, log it in CLAUDE.md so Claude doesn't hit it blind next time:

CLAUDE.md
## Known Issues and Gotchas - Instant DB `useQuery` returns `undefined` on first render -- always check for `isLoading` before accessing data - Tailwind classes don't apply in new components unless the file is in the `content` array in `tailwind.config.js` - Environment variables must be prefixed with `VITE_` to be accessible in the frontend

Or just ask Claude to log it:

claude
That fixed it. Add a note to CLAUDE.md so we don't hit this again.

Common Scenarios

You see... Tell Claude...
White screen"My app shows a blank white page. Check the browser console for errors."
Wrong data showing"The recipe list shows the wrong recipes. Check how data is fetched in [file]."
Styling looks off"The layout is broken on mobile. Check the Tailwind classes in [component]."
Build failsCopy the full build error and paste it.

Troubleshooting

Claude fixes one thing but breaks another

Commit working code before each fix so you can roll back.

The error message is confusing

Just paste it as-is. Claude is good at reading errors even when you can't.

Nothing shows up in the terminal

Check your browser's developer console (right-click → Inspect → Console tab).

Same error keeps coming back

That's a sign it belongs in CLAUDE.md. Ask Claude to add it as a known issue.
