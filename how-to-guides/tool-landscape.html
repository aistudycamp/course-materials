How-To Guide

Which AI Tool Should I Use?

A practical, use-case-first map of the AI tool landscape — organized by what you're trying to do, not by the logo on the box.

How to Use This Guide

Most students get stuck because they pick a tool before they pick a job. The fix is boring but effective: figure out what you're trying to do, then find the section below.

Each section covers one use case, lists 2–4 tools worth knowing, and tells you when each is the right pick (and when it isn't). Tools change fast. The categories don't.

If you're brand new and don't know where to start, skip to the bottom — there's a starter trio.

1. Chatbots & Conversational UIs

Talking to an AI in a chat window. This is the front door for 90% of people and, honestly, still the most useful category for most daily work.

Tool Best for Avoid when
Claude.ai Writing, thinking, analysis, long documents. Projects feature is excellent for recurring context (one client, one book, one course). You need live web search without turning it on, or real-time voice.
ChatGPT Voice conversations, image generation inline, the broadest ecosystem of plugins and custom GPTs. Heavy long-form writing or careful reasoning — Claude tends to feel better here.
Custom via API You're embedding a chatbot into your own product, or need full control over the system prompt and guardrails. You just want to talk to an AI — use the hosted apps, don't build your own.
Student tip

Pick one chat app and actually live in it for two weeks before judging. Most "I don't get AI" problems are "I tried three tools for ten minutes each" problems.

2. Vibe Coding & App Builders

Describe an app in plain English, get a working app back. You barely see code. Great for MVPs, landing pages, and "I just want to see if this idea has legs."

Tool Best for Avoid when
Lovable Full-stack apps with real UI, auth, and a database. Best polish of the category. Our Week 1 tool. You need deep custom backend logic or want to own/host the code yourself from day one.
Bolt.new Throwaway prototypes in seconds. Great for demos and spikes. You want to ship and maintain the app long-term.
Replit Prototyping with a little more control — you can see the code, run servers, learn as you go. You want a purely no-code experience.
v0 (Vercel) Generating React/Next.js components and UIs you'll drop into a real codebase later. You don't already know you want React — v0 assumes that stack.
Student tip

These tools are magic for day one and frustrating by day ten. The moment you find yourself fighting the tool to make a small change, that's your signal to graduate to a coding agent.

3. Coding Agents

AI that reads your whole codebase, edits multiple files, runs commands, and debugs. More power than an app builder, more complexity too. You'll work in a terminal or a real editor.

Tool Best for Avoid when
Claude Code Serious multi-file work, debugging, refactoring, scripting, automation via CLI. Our Week 2 tool and the one you'll use the most. You're allergic to the terminal and refuse to use one.
Cursor (Agent) You want an editor-native experience — see the file, tab-complete, chat in a side panel. Smooth on-ramp from "I've opened VS Code before." You want to run agents from CI, scripts, or across many projects.
Windsurf (Cascade) Similar niche to Cursor. Some students prefer its flow and UI. Worth trying if Cursor didn't click. You've already paid for Cursor and it's working fine.
Codex / Copilot Inline autocomplete inside your editor while you already code. Low-key, always-on assist. You want the AI to own the whole task, not just autocomplete the next line.
Student tip

Don't pick based on vibes — pick based on where you want to live. Terminal-first → Claude Code. Editor-first → Cursor. Don't learn all three. Pick one and get deep.

4. Research & Synthesis

You have a pile of sources — papers, docs, transcripts, PDFs — and you want to understand them, compare them, or answer questions grounded in them. These tools beat a general chatbot because they cite and they don't drift.

Tool Best for Avoid when
NotebookLM Grounded Q&A over your own sources. Bonkers-good audio overviews. Great for book clubs, course prep, legal docs, company knowledge. You need it to pull in fresh info from the open web.
Perplexity Web research with citations. Replaces a lot of "open five tabs, skim, synthesize." You need deep reasoning over your own long documents — NotebookLM or Claude Projects win there.
Claude Projects Ongoing work against a stable set of files (a thesis, a product spec, a legal matter). The context sticks. You want source-by-source citations — the chat format blurs them.
Student tip

If you've never used NotebookLM, go try it right now with 3 PDFs you've been meaning to read. The audio overview alone will sell you.

5. Data & Spreadsheets

Upload a CSV, ask questions, get charts and analysis. Modern AI is genuinely good at this, and it's the category most non-technical people underestimate.

Tool Best for Avoid when
ChatGPT Advanced Data Analysis Drop a CSV or Excel file, get charts, summaries, cleaning, joins. The most mature option. Very large datasets (hundreds of MB) or anything that needs to stay local.
Claude with files Careful, explainable analysis. Good reasoning about what the data means, not just the code. You want polished, styled charts by default — ChatGPT is more opinionated.
Julius Spreadsheet-native analysis with a data-analyst-y UI. Nice if you live in Excel/Sheets already. You're already comfortable in ChatGPT — Julius' moat is thin for power users.
Student tip

Never paste sensitive data (customer info, PII, financials) into a consumer chatbot. Use the API, a local tool, or redact first.

6. Design & UI Mockups

Going from "I have an idea" to "I have something to show." Great for founders, PMs, and anyone who needs a visual to pressure-test an idea before building.

Tool Best for Avoid when
v0 (Vercel) High-fidelity UI components from a prompt. Best when the end state will be real code. You want to stay in a design tool forever — v0's output is code.
Figma Make You already work in Figma and want AI to bootstrap designs inside it. You're not on Figma — don't start there just for this.
Lovable The fastest path from "mock" to "actually clickable app" — the mock runs as a real site. You want a static design artifact to iterate on visually.
Claude with design prompts Single HTML files, slide decks, quick visual artifacts. No stack, no setup, just a file. You need a multi-screen product mock — the tools above do that better.
Student tip

The best mock is one you'd be embarrassed to show. Ship the ugly version to two people before polishing. AI makes this cheap enough that "show it early" beats "get it right."

7. Automation & Workflows

No app UI — these run behind the scenes, connecting tools you already use (Gmail, Slack, Sheets, APIs) and triggering on events.

Tool Best for Avoid when
n8n AI-heavy workflows, complex branching, self-hosting. The most flexible of the category. Our Week 3 tool. You want to connect two SaaS apps in 3 minutes with zero learning curve.
Zapier Simple "when X, do Y" across 5,000+ apps. Fastest onboarding. Your workflow has real branching, loops, or heavy AI — you'll out-grow it.
Make A middle ground — more power than Zapier, more visual than n8n. You already know n8n — there's no reason to switch.
Claude Code + CLI One-off or scripted automations that live on your machine — "every morning, pull these files and summarize." Dev-flavored. Someone non-technical needs to maintain it.
Student tip

Automate something that bugs you weekly, not something impressive. The goal is to feel the magic loop once — then you'll see automation opportunities everywhere.

8. Writing & Content

Drafts, edits, outlines, transcripts into essays, essays into scripts. The oldest use case and still the most underrated one.

Tool Best for Avoid when
Claude.ai Long-form writing, editing, voice-matching. The most "writerly" of the chat tools. You want image/voice generation in the same turn.
ChatGPT Short-form content, social copy, brainstorming in voice mode. Careful, nuanced long-form — Claude tends to have better taste.
NotebookLM Writing from a big pile of sources — research-backed essays, reports, summaries. You're writing from a blank page — it needs sources to shine.
Student tip

Never publish the first draft. The interesting thing AI can do for writers isn't "write the post" — it's "interview you, then help you see what you actually think."

Common Mistakes

Watch out for these

Picking the tool before picking the job

"Which tool should I use?" is the wrong question. Ask "what am I actually trying to do?" first, then look at this guide.

Using an app builder for something that needs a coding agent

Lovable and Bolt are amazing for day one. By day ten, if you're fighting the tool to make small changes, it's time to graduate to Claude Code or Cursor.

Using a coding agent when Zapier would do

If the job is "when X happens, do Y," you don't need to write code. Use automation.

Trying to learn all of them at once

Pick one per category. Get comfortable. You can always switch later — the concepts transfer.

Confusing the product with the model

Claude is the model. Claude.ai, Claude Code, the Claude API, and Lovable (which uses Claude under the hood) are different products. The model is the brain; the product is the interface.

If You're Overwhelmed, Start Here

Too many boxes on this page. Don't worry about it. Here's the starter trio — if you learn these three, you can fake the rest of the landscape.

Tool Why start here
Claude.ai Your thinking partner. Writing, research, analysis, most of your daily AI use. Live here first.
Lovable Your "idea → clickable app in an afternoon" tool. Unlocks the feeling that you can build things.
Claude Code The one that actually changes your career. Worth the discomfort of a terminal.
Rule of thumb

Get 10 hours into each of these before touching anything else on this page. Depth in three tools beats surface in thirty.
