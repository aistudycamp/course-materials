How-To Guide

How to Recover When Things Break

Undo mistakes — ask Claude first, drop to terminal when needed.

Prerequisites

  • Claude Code installed and running — see Install Claude Code
  • A project with at least one Git commit

Before You Panic

Almost nothing is truly lost. Claude Code works with Git underneath, so there are save points to go back to. Take a breath and pick the right approach below.

The Easy Way: Ask Claude

If you're in a Claude Code session, just ask. Claude runs the Git commands for you.

1

Undo the last set of changes

claude
Undo the changes you just made
2

Revert a specific file

claude
Revert App.tsx to how it was before
3

Go back to a specific point

claude
Show me recent commits — I want to go back to before we added authentication.
4

When the whole thing is a mess

claude
Something went wrong — I want to start fresh from the last commit. What should we do?

Claude picks the right Git commands and asks before anything destructive.

Tip

Claude tracks changes it made in the session — "that broke something, undo the last change" is often enough.

The Terminal Way

Outside a Claude session, or doing it yourself:

1

Undo uncommitted changes (safely)

Stashes your changes and reverts files to the last commit. Run git stash pop to bring them back.

zsh
$ git stash $ git stash pop # to bring it back
2

Undo changes to one file

Resets just that file to the last commit.

zsh
$ git checkout -- src/App.tsx
3

Undo a bad commit

Undoes the last commit, keeps your file changes. Fix and commit again.

zsh
$ git reset --soft HEAD~1
4

Go back to an older commit

Find the short hash (e.g. abc1234) from before things broke, then branch from it:

zsh
$ git log --oneline $ git checkout -b recovery abc1234

The Golden Rule

Commit often. More commits = more save points. Commit every time something works — recovery stays easy.

Troubleshooting

Not sure what happened

Ask Claude "show me what changed since the last commit" or run git status.

Claude isn't sure what to undo

Be specific: "undo the changes to App.tsx" rather than "fix this."

Stash seems empty

Check git stash list to see all your stashes.
