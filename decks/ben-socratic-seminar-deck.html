An LLM is a massive mathematical function.

Text goes in. Numbers get multiplied through billions of parameters. Probabilities come out. Everything else is built on top of this.

The model doesn't see words. It sees numbers. Each token maps to an ID.

Common words = 1 token. Longer words split. 1 word ≈ 1.3 tokens. This is why tokens cost money and eat context.

Meaning as geometry

Each token becomes a vector — a point in a 12,000-dimensional space.

Words with similar meanings end up close together. The model "understands" through distance.

How it gets smart

Training happens in three stages.

01Pre-training
Feed it the internet. Hide a word. Ask the model to predict it. Adjust weights. Repeat trillions of times.
Imagine a textbook with random words ripped out. After enough pages, the model can fill in anything.
02Supervised Fine-Tuning
Humans write thousands of example conversations. Model learns the personality of being helpful — how an assistant talks.
Pre-training read every book. SFT showed it how a tutor responds.
03RLHF
Humans rate two responses. Preferred one wins. Do this thousands of times. Model develops judgment.
A manager reviewing work until the new hire develops taste.

This is why Claude and ChatGPT feel different — different RLHF preferences = different personalities.

The memory problem

It has 50 First Dates memory.

Every session is a restart. The model doesn't remember yesterday. You bridge the gap by handing it notes — every time.

Weights
hazy long-term memory
Compressed patterns from training. It "remembers" how Python works — not what you told it last week.
Context window
working memory
What the model can see right now. Your prompt, your files, this conversation. Resets every session.
CLAUDE.md
the notes Adam Sandler writes
You re-inject context at the start of every session. Project docs, skills, memory files — continuity the model can't keep on its own.

Karpathy calls this anterograde amnesia. The whole craft of working with Claude Code is managing which memory you're writing to.

What training leaves behind

The model is a lossy zip file of the internet.

Training doesn't store the internet — it compresses it into patterns. Common patterns survive. Rare details get smudged.

Well-compressed
Python syntax. Common idioms. Writing patterns. Widely-discussed facts. Seen millions of times.
Lossy / smudged
Obscure dates. Niche facts. Precise numbers. Rare names. Seen rarely — compressed away.

Hallucination isn't a bug — it's the compression artifact. When the model hits a gap, it doesn't say "I don't know." It generates the most statistically plausible completion. Sometimes that's right. Sometimes it isn't. The model can't tell the difference.

Why it hallucinates — and how to stop it

It isn't lying.
It's predicting the next token.

A fake citation is just a high-probability pattern: "A 2023 study by Dr. Smith in the Journal of…" — a shape it's seen a thousand times. It can't tell "I remember this" from "this sounds like things I remember."

01
Give it a tool.
Web search. File reads. A calculator. Anything real beats a pattern match.
02
Ground it in context.
Paste the source. Link the doc. Don't ask it to recall — let it read.
03
Verify what matters.
Names, numbers, citations — always check. Specifics are where it drifts.
